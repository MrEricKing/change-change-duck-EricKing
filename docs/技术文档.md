> **项目代号**：路线啵啵机 / 路线改改鸭 / Travel Atlas
> **版本**：v1.0（2026-05）
> **作者**：ZZK + Claude Code
> **定位**：把抖音 / B 站 / 海外短视频里的"种草"，自动拼成可视化、可定制、可分享的旅行海报。

---

## 一、产品概述

### 1.1 一句话定位

> **让博主的"种草"变成你的"成行"**。

输入若干条旅游视频 → AI 自动解析、聚合、补全信息 → 以**可爱可视化地图 + 结构化攻略**呈现 → 对话式 Agent 帮用户**定制专属路线**（人群、预算、强度、想去 / 想避开）。

### 1.2 产品来源

基于 `旅游规划AI产品构思.md` 的 10 小时 MVP 路径，落地后实际功能远超原 MVP 范围。

### 1.3 用户故事

- **场景 A**：刷到 1 条 4 分钟东京 vlog → 上传 → 30 秒内拿到 3 日游攻略（含地铁线路、景点 emoji、避雷点）。
- **场景 B**：上传 1-5 条同城博主视频 → AI 自动取并集 + 去重 + 打分 → 看每条视频对最终攻略的"贡献比例"。
- **场景 C**：生成后输入"删掉武康路加田子坊" → 不需要重传视频，**一句话改路线**。
- **场景 D**：5 日长三角样例（沪 → 苏 → 杭 → 宁 → 锡）一键加载，演示无需 API Key。

## 二、技术栈

### 2.1 总览

| 层级 | 技术 |
|---|---|
| 后端框架 | Flask 3.x（轻量、Python 3.10+） |
| LLM | OpenAI 兼容 Chat Completions（GPT-4o / Claude 3+ / Gemini / Qwen-VL 等多模态模型） |
| ASR | Whisper API（云端，主用）+ faster-whisper（本地兜底） |
| 视频处理 | yt-dlp（下载）+ imageio-ffmpeg（抽帧 / 转码） |
| OCR（可选） | rapidocr-onnxruntime |
| 地图渲染 | Folium + Leaflet |
| 底图瓦片 | 高德 AMap（中国大陆）+ OpenStreetMap（海外）+ Carto Voyager + 高德卫星图 |
| 路径规划 | OSRM 公共 demo（真实路网折线，全球覆盖） |
| 地理编码 | 三级兜底：AMap → LLM 推断 → Nominatim |
| 前端 | 原生 JS + CSS（无框架，~17KB） |
| 字体 | Caveat / Fredoka / ZCOOL KuaiLe（Google Fonts） |
| 部署 | run.bat / run.sh 一键启动，imageio-ffmpeg 自带二进制 |

### 2.2 模型多模态识别

代码内置 `_is_vision_model()` 检测，关键字命中 → 自动启用视觉调用（多张关键帧作为 `image_url` 一并发送）：

```python
keywords = ("4o", "vision", "claude-3", "claude-4", "sonnet",
            "haiku", "opus", "gemini", "qwen-vl", "qwen2-vl",
            "glm-4v", "internvl", "minicpm-v", "yi-vl", "llava")
```

视觉调用失败自动回退纯文本模式，不会让流程中断。

## 四、核心功能模块

### 4.1 视频解析流水线（`pipeline.py`）

```
用户上传视频 (1-5 个)
    ↓
analyze_one_video()  ×N    [new_logic_agent.py]
    ├─ ffmpeg 抽音频 (.wav)
    ├─ Whisper API 转写（长音频自动 60s 切片重试，失败回退 faster-whisper）
    ├─ ffmpeg 抽关键帧 (5-8 张)
    └─ 元数据（标题、tags 等）
    ↓
build_all_fused_text()
    生成 "========== 视频 N ==========" 分段文本
    ↓
call_llm() (多模态)
    ├─ 视觉模型 → text + N 张 image_url 一并发送
    └─ 纯文本兜底
    ↓
optimize_routes()
    贪心最近邻，每天行程不交叉
    ↓
visualize.render_map()
    AMap 瓦片 + OSRM 真实路网 + emoji 贴纸 marker
    ↓
output/guide.json + guide.md + map.html + geocoded_points.json
```

### 4.2 地图渲染（`visualize.py`）

#### 坐标系自动转换
```python
wgs84_to_gcj02(lat, lng)   # GPS → 高德
gcj02_to_wgs84(lat, lng)   # 高德 → GPS
_out_of_china(lng, lat)    # 跳过转换（海外）
```

#### OSRM 路网（替代直线）
```python
fetch_road_polyline(coords_wgs84, profile="driving")
  → OSRM https://router.project-osrm.org/route/v1/...
  → 返回 ~400 个真实路径折点（沿街道、河流走）
```

#### 多图层
- 🇨🇳 高德地图（中国，默认）
- 🌍 OpenStreetMap（海外，默认）
- 🎨 Carto 淡彩（备选）
- 🛰 高德卫星图（备选）

`_is_overseas(points)` 检测 → 默认显示对应底图，用户可右上角 LayerControl 手动切。

#### 站点标记（emoji 贴纸）
```html
<div class="poi-mk" onclick="postMessage(focus-stop,day,idx)">
  <div class="circle">{emoji}<span class="badge">{idx}</span></div>
  <div class="label">{place}</div>
</div>
```
- `icon_anchor=(90, 23)` → emoji 圆心**精确**落在 lat/lng
- 点击向父页发 `postMessage` → 右侧故事手账自动滚动 + 黄色高亮闪烁

#### 跨天路线（彩色文字）
```python
🚄 高铁 → 珊瑚红
✈️ 飞机 → 天蓝
🚇 地铁 → 紫罗兰
🚌 公交 → 青绿
🚶 步行 → 草绿
🚕 出租 → 鹅黄
```
带白色描边，无方框，模糊的瓦片底也清晰。

### 4.3 多视频贡献条（`server.py`）

```python
_extract_video_sections(fused)  # 解析 "========== 视频 N ==========" 分段
_collect_plan_places(plan)       # 收集 plan 中所有 place 名
_round_percentages(scores)       # 100% 配额分配（floor + 余额排序补偿）
_ensure_video_contributions(plan)
```

算法：每个视频段对最终 plan 地点的命中数 × 3 + 文本长度归一化 → 归一为百分比。

UI 显示：底栏每个视频一根进度条 + 命中地名 tooltip。

### 4.4 三级地理编码（`geocode.py`）

```
缺 lat/lng 的 stop
    ↓
高德 AMap API（设 AMAP_KEY） → 国内最准
    ↓ 失败
LLM 推断（同 API Key） → 全球可用
    ↓ 失败
Nominatim OSM → 兜底
    ↓
缓存到 output/geocache.json
```

## 八、运行 & 部署

### 8.1 一键启动

**Windows**：双击 `run.bat`
**Mac / Linux**：`bash run.sh`

脚本逻辑：
1. 检查 venv 是否存在 → 不存在则 `python -m venv .venv`
2. 检查 flask 是否安装 → 缺则 `pip install -r requirements.txt`
3. 3 秒后自动开浏览器 `http://127.0.0.1:5000`
4. 启动 `python server.py`

### 8.2 环境变量

| 变量 | 默认 | 说明 |
|---|---|---|
| `VECTRUST_API_KEY` / `OPENAI_API_KEY` | — | LLM + Whisper |
| `OPENAI_BASE_URL` | `https://api.openai-next.com` | 接口端 |
| `OPENAI_MODEL` | `gpt-4o-mini` | 模型名 |
| `AMAP_KEY` | — | 高德 Web 服务 Key（可选） |
| `HOST` / `PORT` | `127.0.0.1` / `5000` | 监听 |

### 8.3 ffmpeg 自动化

`server.py` 启动时：
1. 把 `.venv/Scripts/` 加入 PATH（yt-dlp.exe 能找到）
2. 调 `imageio_ffmpeg.get_ffmpeg_exe()` 拿二进制
3. 复制到 `.venv/Scripts/ffmpeg.exe`（让 pipeline 里的 `shutil.which("ffmpeg")` 命中）

**结果**：用户不需要单独安装 FFmpeg，pip 装好 imageio-ffmpeg 即可。

## 十一、已知问题 / 后续可优化

1. **OSRM 公共服务限速**：免费 demo 偶尔超时，已加缓存 + 失败回退直线
2. **抖音直链下载**：抖音反爬严，建议用户用 snaptik.app 等下载后上传本地文件
3. **海外覆盖**：高德海外几乎空白 → 自动切 OSM；OSRM 全球可用所以路线没问题
4. **国内 OCR**：rapidocr 可选未默认启用，OCR 失败不影响主流程
5. **多视频 token 上限**：5 个视频 + 关键帧总 token 可能超出小模型上下文 → 已对 fused 文本做长度截断
6. **API 成本**：Whisper API 长视频按时长计费，本地 faster-whisper 兜底节省成本
7. **真实路网精度**：OSRM 用 OSM 数据，中国大陆部分小路覆盖不全（高德 Directions API 更准但需付费 Key）
